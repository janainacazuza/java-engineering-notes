# Maintenance Ritual: How This Vault Does Not Rot

Vaults without ritual rot. Three rituals, total cost approximately 30 minutes per week.

## Weekly. Inbox cleanup (Friday, 15 min)

Open `99_inbox/`. For each note:
1. Apply the actionability filter.
2. If it passes: move to the correct folder, complete frontmatter, create links following graph rules.
3. If it does not pass: delete.

Non-negotiable rule: inbox leaves Friday empty.

## Monthly. "Why" audit (1st of the month, 10 min)

Open 5 random notes. For each one, re-read the section "Why this changes how I write code". Does it still hold? Is it still true? If not: refactor or delete.

The healthy vault shrinks when it is not growing. Removed notes are a sign of judgment, not loss.

## Quarterly. Scheduled revisit (1st of the quarter)

List notes with `revisit_on` due this quarter. Each one is re-read in a dedicated deep-work session:
- Confirm what is still true.
- Update what changed.
- Update `date_revised` and set new `revisit_on`.

## Signs of a healthy vault

- Empty inbox every Friday.
- Notes being deleted regularly.
- Graph with dense clusters and thin bridges.
- Each note answers a real interview question.
- When you search for something specific, you find it in under 30 seconds.

## Signs of a rotting vault

- Inbox with more than 10 items for over a week.
- Notes without "Why this changes how I write code".
- Graph turning into a uniform cloud without clusters.
- Duplicate notes in different folders.
